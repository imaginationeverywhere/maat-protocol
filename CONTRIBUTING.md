# Contributing to Maat Protocol

Thank you for your interest in contributing to Maat Protocol.

## How to Contribute

### Documentation
- Add examples for LLM providers we haven't covered
- Share real-world case studies of multi-agent orchestration
- Improve existing documentation for clarity
- Translate documentation to other languages

### Templates
- Create manager loop templates for specific providers
- Build escalation rule templates for common scenarios
- Share configuration patterns that worked for your team

### Examples
- Provider-specific orchestration examples
- CI/CD pipeline integration patterns
- Use case walkthroughs with real metrics (cost savings, time saved)

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b add-openai-example`)
3. Make your changes
4. Submit a pull request with a clear description

## Code of Conduct

Be respectful. Be constructive. Focus on making AI agent orchestration better for everyone.

## Questions?

Open an issue. We're here to help.
