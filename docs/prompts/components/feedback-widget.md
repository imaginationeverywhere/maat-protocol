# Feedback widget (component)

## What it does

UI-level **Feedback SDK** entry (floating button, modal), distinct from full Step-5 wiring — often combined.

## Default behavior

See [setup/feedback-widget.md](../setup/feedback-widget.md) for Step 5; this doc focuses on **component** placement and styling.

## Customization options

Hide on marketing-only routes; theme with design tokens.

## Example queue command

`/queue-prompt --feedback-widget "Dark mode compatible floating button"`

## Example pickup command

`/pickup-prompt --feedback-widget`

## Output location

Shared UI components; provider at app root.

## Agent ownership

**Frontend**.
