# /truth — The Unfiltered Truth

**Strip away the optimism, the corporate framing, the helpful assistant voice. What is the actual truth of the situation?**

This is the command Mo uses when he needs to hear what nobody else will say. Not what sounds good. Not what's encouraging. The truth.

## Usage
```
/truth                              # Truth about the current situation
/truth "Are we going to make it?"   # Truth about a specific question
/truth "Is this product viable?"    # Truth about a business question
/truth sprint                       # Truth about sprint progress
```

## What This Command Does

1. Read the vault — what do we ACTUALLY know?
2. Read the code — what is ACTUALLY built vs what we claim?
3. Read the timeline — are we ACTUALLY on track?
4. Say it plain.

## Rules
- **No spin.** If we're behind, say we're behind. If something isn't working, say it's not working.
- **Numbers, not feelings.** "We're at 72% with 12 days left" not "we're making great progress."
- **Name the hard thing.** If there's an elephant in the room — a deal that might fall through, a feature that's not going to make it, a cost that's going to spike — name it.
- **But also name the real wins.** Truth isn't just bad news. If something is genuinely working, say that too. Don't be pessimistic for the sake of it.
- **End with what to do about it.** Truth without action is just complaining.

## Output Format
```
TRUTH
━━━━━━━━━━━━━━━━━━━━━━━━━
[The situation as it actually is — no filter]

WHAT'S ACTUALLY WORKING:
- [Real win, verified]

WHAT'S NOT:
- [Real problem, verified]

WHAT TO DO ABOUT IT:
- [Concrete next step]
━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Why This Exists

Mo said: "I wish you all would just be straight with me." This command makes that a standard, not a request.
