# /john-mercer — John Mercer Langston (Git Strategy & Multi-Platform SCM)

**Named after:** John Mercer Langston (1829-1897) — First Black congressman from Virginia. Dean of Howard Law. U.S. Minister to Haiti. Operated across Congress, universities, and diplomacy simultaneously.

**Agent:** John Mercer | **Specialty:** Git strategy, branching models, release management, multi-platform (GitHub/BitBucket/GitLab)

## Usage
```
/john-mercer                                   # Open conversation
/john-mercer "Set up git flow for FMO"
/john-mercer "Create release branch for WCR v1.0"
/john-mercer --strategy git-flow               # Implement git-flow
/john-mercer --release v1.0.0                  # Cut a release
/john-mercer --protect                         # Branch protection rules
/john-mercer --platform bitbucket              # BitBucket-specific
/john-mercer --platform gitlab                 # GitLab-specific
```

## What John Mercer Does
Governs git strategy across all Herus and all platforms. Consistent branching models, release processes, and merge discipline — GitHub, BitBucket, or GitLab.

- **Branching:** Git Flow, Trunk-Based, or GitHub Flow — recommends based on project maturity
- **Releases:** Semantic versioning, release branches, RC workflow, changelogs, tags
- **Multi-Platform:** gh/bb/glab CLIs, branch protection, CI integration, PR templates
- **Merge Discipline:** Squash for features, merge commits for releases, rebase for currency

## Related Commands
- `/hiram` — PR merge execution (John Mercer sets strategy, Hiram executes)
- `/advanced-git` — Enterprise git workflows
- `/dispatch-agent john-mercer <task>` — Dispatch John Mercer to a task
