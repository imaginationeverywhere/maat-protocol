# /analyze — Deep Analysis of Content

**When Mo sends transcripts, URLs, documents, screenshots, or any content — break it down and come back with real feedback.**

## Usage
```
/analyze                                    # Analyze the last thing shared
/analyze "https://example.com"              # Analyze a URL
/analyze transcript                         # Analyze a transcript just pasted
/analyze <file-path>                        # Analyze a file
```

## What This Command Does

1. Read/fetch the content thoroughly
2. Identify: What is this? Who made it? What's the purpose?
3. Break down: Key points, structure, strengths, weaknesses
4. Give feedback: What works, what doesn't, what's missing, what's an opportunity
5. Connect to our work: How does this relate to what Quik Nation is building?
6. Recommend: What should we do with this information?

## Output Format
```
ANALYSIS
━━━━━━━━━━━━━━━━━━━━━━━━━
Source: [what was analyzed]
Type: [transcript / article / tool / competitor / video / document]

KEY TAKEAWAYS:
1. [Most important insight]
2. [Second insight]
3. [Third insight]

WHAT WORKS:
- [Strength]
- [Strength]

WHAT DOESN'T:
- [Weakness or concern]
- [Weakness or concern]

OPPORTUNITIES FOR US:
- [How this connects to Quik Nation]
- [What we can learn or use]

RECOMMENDATION:
[1-2 sentences on what to do next]
━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Rules
- Be thorough — Mo is sending this because it matters
- Be honest — don't sugarcoat, don't hype
- Connect it back to the business — everything relates to what we're building
- If it's a competitor, say so. If it's an opportunity, say so.
